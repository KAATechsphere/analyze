# Нефункциональные требования

1. **Производительность**
   - Время отклика на запрос пользователя составляет не более 2 секунд при нормальной нагрузке и не более 5 секунд при максимальной нагрузке.
   - Время обработки и сохранения данных о тренировке не превышает 3 секунд.

2. **Безопасность**
   - Применяется шифрование трафика.
   - Регулярные проверки на наличие уязвимостей и своевременное их устранение.

3. **Масштабируемость**
   - Инфраструктура готова поддержать увеличение количества пользователей на 50% без падения производительности.
   - Система поддерживает горизонтальное масштабирование для обработки возросшей нагрузки.

4. **Удобство использования**
   - Процент пользователей, сталкивающихся с трудностями при выполнении стандартных операций, не превышает 5%.
   - Интерфейс приложения интуитивно понятен и удобен для большинства пользователей.

5. **Доступность**
   - Уровень доступности системы не менее 99,9%, время восстановления после сбоев не превышает 15 минут.
   - Система имеет резервные копии данных и механизмы автоматического восстановления.

6. **Совместимость**
   - Поддерживаемые платформы: Android 7+ и iOS 11+.
   - Поддерживаемые браузеры: Chrome, Safari, Firefox последних версий.
   - Приложение поддерживает интеграцию с популярными фитнес-браслетами.

7. **Документируемость**
   - Документирование всех значимых компонентов и процессов, покрывает не менее 90% функционала.
   - Документация регулярно обновляется и доступна для всех членов команды.
